# UI/UX Standard Checklist (Ver.3.0)

Version: 3.0
Effective date: 2022-11-21

This checklist captures the minimum UI/UX consistency requirements for internal
web-based systems.

## Breakpoints

- PC: `>= 1024px`
- Tablet: `768px - 1024px`
- Phone: `< 768px`

## Layout

- No horizontal scroll at recommended resolutions.
- Phone defaults to single-column, content-first layout.
- Tablet may use 2-pane layouts when it improves navigation (for example,
  thumbnail list + preview canvas).

## Forms

- Required fields are marked with `*` and the meaning is consistent across the
  entire page.

## Buttons and labeling

- Use verb-first labels.
- Keep ordering consistent across screens whenever the actions are equivalent.
- Avoid showing operator-only actions to end users unless they are guaranteed to
  work in the current auth model.

## Color token

- Base brand color token: Hyosung Blue `#034ea2`.
